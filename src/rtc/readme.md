
# RTC KPI

## 功能介绍
- 统计各成员的修复Bug数量
- 统计结果输出报告
- 统计数据保存, 可基于初始数据和最后一次的统计数据计算两次统计周期期间新增问题数量



## 功能模块的责任
- config  - 解析配置文件
- htmlloader： 加载html内容，来源：从本地、网络
- parser: 解析html并提取数据进行统计
- db: 统计数据的加载和持久化
- report_output: 输出统计结果，格式：文本/Markdown ， `注`：文本格式输出同时会输出在日志文件
- RtcMain: 程序入口 ，

## 配置文件和输出报告格式模板

> 配置文件：
```json
{
    "version":"2",
    "members":["A","B","C"],
    "report_title": "X团队 修复问题汇总",
    "report_fmt":"text",
    "rtc":[
        {
            "url":"file:///",
            "element_id_1":"table1",
            "element_id_3":"table2"
        },
        {
            "url": "http://",
            "element_id_1":"table1",
            "element_id_3":"table2"
        }
    ]
}
```


> 输出结果数据结构：
```json
{
    "date":"2020-1-15",
    "A":{
        "fixed":"90",
        "out":"1"
    },
    "B":{
        "fixed":"62",
        "out":"1"
    }
}
```


> 初始数据的格式
```json
#初始数据和最后一次的统计数据保存在一个文件里
{
    "date":"2019-10-15",
    "A":{
        "fixed":"3",
        "out":"1"
    },
    "B":{
        "fixed":"2",
        "out":"1"
    }
}
```
